Does using relative size matter?

In my opinion, yes it matters to use relative size like %, em or rem when styling the size of an element. I say this because of the following reasons.

First, we may only want to size a specific element with respect to the size of its parent box. In those cases, using em or rem will be beneficial for us. Another reason is when we think about our browsers zoom feature. It is because of the relative size feature that we are able to zoom things on our browsers without changing the actual size of the browser.

All in all, using relative size is advantageous because they offer responsiveness, scalability, improved reading experience, and greater flexibility in defining components.

Meaning and Interpretation of 200px

As we can see from its unit, the value 200px has a fixed (absolute) va;ue of 200 unit screen pixels to take place for whatever the object/element it is assigned for. It would have an effect if we had used relative measuring units such as %, rem and em.

For example, if we use 200% for the font-size property of a text, we will get a 32px (2 times that of the default 16px value) since we are referring with respect to the root width.

The Specific Changes in CSS3

CSS3 is the latest evolution of the Cascading Style Sheets language and aims at extending CSS2. It brings a lot of new features and additions.
These changes in technical terms are listed below.

1. Selectors

CSS3 added some new useful and important selectors.
[attr^=val]: matches an element with the attribute attr and a value starting with val
[attr$=val]: matches an element with the attribute attr and a value ending with the suffix val
[attr*=val]: matches an element with the attribute attr and a value containing the substring val

2. Color

In CSS3 RGBA color assignment was added. RGBA works just like RGB, except that it adds a fourth value: alpha, the opacity level or alpha transparency level.
Example: rgba(0, 0, 255, 0.5) would give me a 50% transparent blue color.

3. Rounded-Corners

CSS3 added a property called border-radius that lets us create rounded images without any outside help. We simply add border-radius: #px; to round our images.

4. Border Image

Another new feature in CSS3 is the ability to swap out a border with an image. The property border-image allows us to specify an image to display instead of a plain solid-colored border.

5. Drop Shadows

CSS3 provides the ability to add drop shadows to elements using the box-shadow property.
A possible example is box-shadow: 2px 5px 0 0 rgba(72,72,72,1);

6. Text Shadow

The new text-shadow property allows us to add drop shadows to the text on a webpage.
Example: text-shadow: topOffset leftOffset blurRadius color;

7. Linear Gradients

In CSS3, the syntax for generating linear gradients was added.
Syntax - background: linear-gradient(direction, color-stop1, color-stop2, ...);
Example: background: linear-gradient(to right, red , yellow);

8. Transform (Element Rotation)

CSS 3 also introduced a property called transform , which enables rotating Web elements on a webpage.
-moz-transform: rotate(90deg); This will rotate the element by 90deg
-webkit-transform: rotate(180deg); This is also possible

9. Web Fonts

CSS 3 also facilitates embedding any custom font on a webpage. By using the @font-face property, we can include a font from a remote location and use it.
Example: @font-face {
       font-family: myfont_bold;
       src: url(“myfont.ttf”);
     }

10. Multicolumn Layout

In CSS 3, by using the multicolumn layout property we can specify the number of columns we need and they will be created.
-moz-column-count: 3; This will create a 3 column webpage.
